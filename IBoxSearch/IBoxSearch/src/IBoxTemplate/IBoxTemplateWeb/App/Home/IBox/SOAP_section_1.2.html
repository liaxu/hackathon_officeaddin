Standard text and formatting

Follow all authoring guidelines described in General Authoring Guidelines.

The following boilerplate text should appear in section 1.2:

References to Microsoft Open Specifications documentation do not include a publishing year because links are to the latest version of the documents, which are updated frequently. References to other documents include a publishing year when one is available.

In other sections of a specification, normative and informative references should be cited in one of the ways shown in the following table.

Type of citation

Description of type

Correct language

Example

General normative

The entire technical document must be read.

"as specified in [citation]"
- or -
"For more details, see [citation]"

"an absolute uniform resource identifier (URI), as specified in [RFC3986]."

Specific normative*

Only a specific section of the technical document must be read.

"as specified in [citation] section #"
- or -
"For more details, see [citation]."

"as specified in [ECMA-335] Partition I section 8.5.2."

General informative

The entire technical document is of interest but is not required.

"as described in [citation]"
- or -
"For more information, see [citation]."

"For more information, see [MSDN-FONTS]."

Specific informative*

Only a specific section of the technical document is of interest but is not required.

"as described in [citation] section #"
- or -
"For more information, see [citation] section #."

"For more information, see [MS-OLEDS] section 2.2."

Other normative

Other language that clearly indicates normative content.

"as defined in [citation]"

"This section contains common definitions used by this protocol. The syntax of the definitions uses XML schema as defined in [XMLSCHEMA1] and [XMLSCHEMA2], and WSDL as defined in [WSDL]."

Other informative

Other language that clearly indicates informative content.

See example.

"While this information is not necessary to implement the protocol, the information contained in [RFC3066] is helpful background information on the best practices for tags for the identification of languages."

* Specific citations should always provide the applicable section number in the reference, in addition to the short name of the reference technical document. Do not, however, request separate reference entries for specific sections of content. Instead, cite the section number with the linked reference short name in the protocol specification text.
