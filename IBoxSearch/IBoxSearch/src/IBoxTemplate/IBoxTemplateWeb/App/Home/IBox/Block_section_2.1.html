Standard text and formatting

Follow all authoring guidelines described in General Authoring Guidelines. For additional information on standard text and formatting for this section, see the standards listed at the end of the Standard text and formatting for section 2 Messages.

Required elements

  • Provide an overview of how protocol messages are encapsulated and what those messages contain

  • Explain the security model and security parameters that are used

  • For SOAP-based protocols, specify which versions of SOAP and what SOAP specifications are used

  • Use normative terms, as specified in RFC 2119, wherever possible

Inappropriate elements

  • Lowercase versions of normative terms; for example, must, should, and may

  • Diagrams, figures, or other art

  • Footnotes

  • Code examples

  • Pseudocode

  • Product names, unless those names are part of a protocol name

Checklist

  • Describes how protocol messages are transported and any required parameters

  • Explains the security model and security parameters that are used

  • Provides links to more detailed information, as necessary, in subsequent sections of the specification

  • For SOAP-based protocols, specifies which versions of SOAP and what SOAP specifications are used

  • Normative terms, as specified in RFC 2119 are used and capitalized correctly

  • Does not use the lowercase versions of normative terms

  • Does not use the technical document short name ([MS-xxxx]) to refer to the protocol

  • Normative and informative citations cite only approved references

  • Presents product behavior notes in the correct format <#> and links to the correct note in the Product Behavior appendix

  • Uses the correct product name in references to specific products

  • Does not contain pseudocode or code examples

  • Does not contain footnotes

  • Does not contain diagrams, figures, or other art
