Standard text and formatting

Follow all authoring guidelines described in General Authoring Guidelines.

If there are no higher-layer triggered events for the protocol, do not delete this section. Instead, ensure that the first and only sentence in this section is "None."

Required elements

  • If there are no higher-layer triggered events for the protocol, ensure that the first and only paragraph in this section is "None."

  • Provide one section per trigger.

  • Describe events that are triggered by a higher layer.

  • Specify actions that are taken in response to triggered events.

  • Specify behaviors that occur in response to triggered events.

  • Organize behavior requirements by what triggered the behavior, not by action.

Inappropriate elements

  • Information about actions that are taken when the protocol starts, independent of any parameters specified by a higher-layer protocol

  • Triggered event information that is not relevant to the current section 3.x

  • Code examples

  • Pseudocode

  • Lowercase versions of normative terms; for example, must, should, and may

  • Product names, unless those names are part of a protocol name

Checklist

  • If there are no higher-layer triggered events for the protocol, the first and only paragraph in this section is "None."

  • Specifies only those events that are triggered by a higher layer.

  • Provides one section per trigger.

  • Specifies actions that are taken in response to triggered events.

  • Specifies behaviors that occur in response to triggered events.

  • Organizes behavior details by trigger, not action.

  • Does not include initialization actions that are taken independent of any higher-layer events.

  • Specifies only those behaviors that are applicable to the role defined in the same section 3.x.

  • Does not contain code examples or pseudocode.

  • Normative terms, as specified in RFC 2119, are used and capitalized correctly.

  • Does not use the lowercase versions of normative terms.

  • Normative and informative citations cite only approved references.

  • Presents product behavior notes in the correct format <#> and links to the correct note in the Product Behavior appendix.

  • Uses the correct product name in references to specific products.

  • Provides a figure number and caption for each diagram, figure, or graphic.

  • Does not use the technical document short name ([MS-xxxx]) to refer to the protocol

  • Footnotes are only used if you are not authoring in GenDox; any footnote text appears immediately after the table and clearly indicates which information it applies to.

  • Does not contain product names, unless those names are part of a protocol name.
