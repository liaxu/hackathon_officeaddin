Standard text and formatting

Follow all authoring guidelines described in General Authoring Guidelines.

If no timers are used, do not delete this section. Instead, ensure that the first and only sentence in this section is "None." This is frequently the case for server protocols in which requests are initiated by protocol clients.

If timers are used, timer descriptions should be formatted as follows:

A/An/The <timer name> + <third-person, present-tense verb such as "measures"> + <brief description that forms a complete sentence>.

For example:

The ConnectionEstablishment timer measures the time that it takes for a connection to move from the connecting state to the established state.

Required elements

  • If the protocol does not require any timers, ensure that the first and only paragraph in this section is "None."

  • Describe any timers that are inherently required by the protocol, independent of the implementation.

  • Specify the default value and legal range of values for each timer.

  • If the protocol uses non-default behavior for timers in the underlying transport protocol, specify those times and constraints.

  • Ensure that all of the states that are specified in this section are also mentioned elsewhere in the specification when specifying the behavior of the protocol.

Inappropriate elements

  • Prescriptive implementation details

  • Information about timers that are not relevant to the current section 3.x

  • Lowercase versions of normative terms; for example, must, should, and may

  • Diagrams, figures, or other art

  • Code examples

  • Pseudocode

  • Product names, unless those names are part of a protocol name

Checklist

  • If the protocol does not require any timers, the first and only paragraph in this section is "None."

  • Lists all applicable timers and explains why they exist.

  • Includes default values and/or legal ranges of values for each timer, if applicable.

  • Does not prescribe specific timer implementations.

  • Does not provide implementation-specific details.

  • Describes only those timers that are specified in subsections within the same section 3.x.

  • Describes timers using the following format: A/An/The <timer name> + <third-person, present-tense verb such as "measures"> + <brief description that forms a complete sentence>.

  • Normative and informative citations cite only approved references.

  • Presents product behavior notes in the correct format <#> and links to the correct note in the Product Behavior appendix.

  • Normative terms, as specified in RFC 2119, are used and capitalized correctly.

  • Does not use the lowercase versions of normative terms.

  • Does not use the technical document short name ([MS-xxxx]) to refer to the protocol.

  • Does not contain pseudocode or code examples.

  • Footnotes are only used if you are not authoring in GenDox; any footnote text appears immediately after the table and clearly indicates which information it applies to.

  • Does not contain diagrams, figures, or other art.

  • Does not contain product names, unless those names are part of a protocol name.
