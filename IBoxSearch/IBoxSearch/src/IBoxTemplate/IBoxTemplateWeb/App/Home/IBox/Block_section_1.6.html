Standard text and formatting

Follow all authoring guidelines described in General Authoring Guidelines.

None.

Required elements

  • Explain the applicability or design intent and the applicability limits or design limitations.

  • Describe significant system requirements and configurations that facilitate or impair use of the protocol, algorithm, structure, or file format.

  • For protocols, indicate when the protocol is and is not appropriate for use by upper-layer protocols.

  • For algorithms, indicate when the algorithm is and is not appropriate for use by protocols or other algorithms.

  • For structures, document any assumptions, preconditions, initial state of the system, or underlying protocols or structures that must exist before this structure is populated or interpreted.

  • For structures, indicate what must be in place on the system, and what information the client and server must recognize, for the structure to be populated or interpreted properly.

  • For structures, further describe the dependencies described in Overview.

Inappropriate elements

  • Normative terms, as specified in RFC 2119; for example, MUST, SHOULD, and MAY

  • Lowercase versions of normative terms; for example, must, should, and may

  • Normative citations

  • Product behavior notes

  • Footnotes

  • Diagrams, figures, or other art

  • Tables

  • Code examples

  • Pseudocode

  • Product names, unless those names are part of a protocol name

Checklist

  • Explains the design intent and design limitations of the protocol, algorithm, structure, or file format

  • Identifies significant, relevant system requirements and configurations

  • Describes appropriate usage by higher-layer protocols

  • For algorithms, describes appropriate usage by protocols and other algorithms

  • For structures, describes system requirements or information that is needed for the structure to be populated or interpreted properly

  • Further describes the dependencies described in Overview.

  • Does not use normative terms, as specified in RFC 2119

  • Does not use the lowercase versions of normative terms

  • Does not use the technical document short name ([MS-xxxx]) to refer to the protocol

  • Uses informative citations sparingly

  • Does not contain normative citations

  • Does not contain pseudocode or code examples

  • Does not contain footnotes or product behavior notes

  • Does not contain diagrams, figures, or other art

  • Does not contain tables

  • Does not contain product names, unless those names are part of a protocol name
