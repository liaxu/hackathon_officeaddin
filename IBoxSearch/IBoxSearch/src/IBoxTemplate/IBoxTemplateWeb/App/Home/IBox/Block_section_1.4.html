Standard text and formatting

Follow all authoring guidelines described in General Authoring Guidelines.

REST, RPC, and Block protocols should include a protocol stack diagram when the protocol depends on other protocols. Create a protocol diagram based on the SOAP suggestions that follow.

For SOAP protocols that are based on HTTP, the following text and diagram are suggested.

This protocol uses SOAP over HTTP, as described in [RFC2616], as shown in the following layering diagram.

For SOAP protocols that are based on HTTPS, the following text and diagram are suggested.

This protocol uses SOAP over HTTPS, as described in [RFC2818], as shown in the following layering diagram.

For SOAP protocols that are based on HTTP or HTTPS, the following text and diagram are suggested.

This protocol uses SOAP over HTTP, as described in [RFC2616], and SOAP over HTTPS, as described in [RFC2818], as shown in the following layering diagram.

For SOAP protocols that are not based on HTTP, replace the appropriate text in the preceding description and diagram with the correct text.

Exchange: In Exchange Web Services SOAP protocol documents, provide information about the Autodiscover protocol that is used to determine the end point to target for Web service requests. The following are the Autodiscover protocols:

  • Autodiscover Publishing and Lookup SOAP-Based Web Service Protocol [MS-OXWSADISC]

  • Autodiscover Publishing and Lookup Protocol [MS-OXDSCLI]

Required elements

  • Identify any dependencies on lower-layer protocols.

  • Identify any interactions with parallel protocols.

  • Identify any protocols that depend on this protocol.

  • Identify any relationships between this protocol and other protocols.

Inappropriate elements

  • Exhaustive descriptions of other protocols that depend on this protocol

  • Normative terms, as specified in RFC 2119; for example, MUST, SHOULD, and MAY

  • Lowercase versions of normative terms; for example, must, should, and may

  • Normative citations

  • Product behavior notes

  • Footnotes

  • Tables

  • Code examples

  • Pseudocode

  • Product names, unless those names are part of a protocol name

Checklist

  • Identifies any dependencies on lower-layer protocols or other protocols

  • Identifies any interactions with parallel protocols

  • Identifies any relationships with other protocols

  • Identifies other protocols that depend on this protocol

  • Includes the appropriate diagrams, as necessary

  • Provides a figure number and caption for each diagram; for example, "Figure 1: This protocol in relation to other protocols"

  • Does not use normative terms, as specified in RFC 2119

  • Does not use the lowercase versions of normative terms

  • Does not use the technical document short name ([MS-xxxx]) to refer to the protocol

  • Uses informative citations sparingly

  • Does not contain normative citations

  • Does not contain pseudocode or code examples

  • Does not contain footnotes or product behavior notes

  • Does not contain tables

  • Does not contain product names, unless those names are part of a protocol name
